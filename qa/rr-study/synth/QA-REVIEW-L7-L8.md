## Blocking — must clear before merge

### F-101 — Clean-room mandate breached → **re-derive from the QEX paper** (Captain's ruling, 2026-06-06)

DEV-BRIEFING §1 (non-negotiable) and STUDY-SPEC §5 require deriving packing and LDPC from the
**published protocol description** (Franke/Somerville/Taylor, QEX 2020) — **not** porting from
`ft8_lib`. As submitted:

- Provenance comments throughout `packing.py` / `ldpc.py` cite **`ft8_lib` source files**
  (`message.c`, `constants.c`, `text.h`) and the `.f90` reference programs — not paper sections.
- `_normalize_to_c6` / `_pack_basecall` are documented as *"replicating the normalisation logic
  from pack_basecall (ft8_lib/ft8/message.c)"* — that is porting algorithm, which §1 forbids.
- `native/ft8_lib` (the library OpenWSFZ itself was ported from in p12) is checked out in the
  working tree.

**Why this is load-bearing, not pedantry:** OpenWSFZ — the decoder under test — is itself a port of
`ft8_lib`. If the synthesiser is *also* derived from `ft8_lib`, the oracle and the instrument share a
common ancestor; a latent `ft8_lib` encoding defect would live in both and the harness would mask the
exact class of defect §5 exists to expose.

**Required:**
1. Re-transcribe `packing.py` and `ldpc.py` from the **QEX paper**, citing paper sections/tables in
   every provenance comment. Remove all `ft8_lib`-path citations.
2. Published **constant tables** (LDPC generator & H matrices, character alphabets, CRC polynomial,
   Costas array) may remain *if* re-sourced and re-cited to the paper — they are protocol facts.
3. The callsign-packing **logic** (`_normalize_to_c6`, `_pack_basecall`, special-token handling) must
   be original code derived from the paper's field definitions. Strike all "replicating ft8_lib"
   language.
4. **Remove `native/ft8_lib` from the working tree** for this work so it cannot serve as a crib.

### F-102 — "Bit-exact" worked example is self-referential, not independently sourced

DEV-BRIEFING §3.4.2 requires a vector *"taken from the published protocol material, not generated by
ft8_lib"* and *"coordinate with QA so we agree the reference is independent."* The current
`_CQ_Q1ABC_FN42_EXPECTED_INT` is computed **in the test file using the same mixed-radix formula and
offset constants as the implementation**. A systematic transcription error (wrong `NTOKENS`,
mis-ordered character table) would corrupt the implementation and the "expected" value identically and
the test would still pass — it verifies internal consistency, not correctness.

**Required:** assert against a genuinely external published 77-bit / 174-bit / tone vector, and run
the chosen reference past QA so we jointly confirm independence before it lands.

DEV-BRIEFING §6: mark L7/L8/L9 ✅ *only* after the §5 gate is green. L7/L8 correctly show 🔶; L9 shows
✅ "done". Set L9 to "implemented, gate pending" for consistency.

### F-105 — Dead directed-CQ branch in `_pack_callsign` *(Low / cosmetic)*
The `"CQ NNN"` branch (`startswith("CQ ") and len == 6`) is unreachable: `pack_message` splits on
whitespace, so a callsign token can never contain a space. It is also out of §3.2 scope. Remove it or
mark it explicitly reserved — as written it implies support that does not exist.
