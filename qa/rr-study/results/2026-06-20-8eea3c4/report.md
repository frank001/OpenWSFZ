## Section 1 — Study Hypothesis

**Purpose of this run:** Targeted S5-only verification that the D-009 R2 fix (commit
`8eea3c4`, shim 20260026) reduces the OSD-induced S5 false-positive rate below the gate
threshold of 6.0%, following the R1 text-only filter's failure at 133.3% (`d40b4cd`,
2026-06-20).

**Background:** The OSD fallback introduced at shim 20260025 (`fix/d001-osd-fallback`,
merged `1809ce7`) closed the D-001 co-channel decode gap but also manufactures
structurally valid FT8 messages from pure AWGN via CRC-14 coincidence. The full S1–S8
gate run at `6e821fa` (2026-06-20) measured S5 FP rate = 91.67% and raised defect D-009.
The R1 filter (`d40b4cd`) added three text-layer guards (D9-R1/R2/R3) to
`Ft8Decoder.IsPlausibleMessage` but was verified insufficient — 133.3% event FP rate,
with every residual FP having a base callsign ≤ 6 characters, the exact D9-R3 boundary.
`d40b4cd`'s Section 5 concluded that **text-level heuristics have a hard ceiling** and
prescribed a native OSD confidence gate as the required R2 fix.

**D-009 R2 fix (commit `8eea3c4`):**
- **Native OSD correlation gate** in `decode.c` (both `ftx_decode_candidate` and
  `ftx_decode_candidate_ap` call sites): after a successful OSD CRC-14 hit, computes
  `score = Σ(hard_pm1[i]·llr[i]) / Σ|llr[i]|` over the 174 codeword bits and rejects the
  candidate when `score < OSD_CORR_THRESHOLD` (initial value 0.10).
- **Text-layer Gap A fix:** `IsCallsignOversized` now applied unconditionally to both
  tokens of a 2-token message (previously only checked token1 when token0 was CQ/DE/QRZ).
- **Text-layer Gap C fix:** removed the `text.StartsWith("CQ ")` early-return from
  `IsPlausibleMessage`, so 3-token CQ messages fall through to the same grid/report
  validation as standard QSO messages.

**Null hypothesis under test:**

- **H₀_D9R2:** The R2 fix (native correlation gate + Gap A/C text extensions) reduces
  the S5 FP rate to ≤ 6.0%.
  Rejection criterion: S5 FP rate (OpenWSFZ) > 6.0% on the same 12-slot S5 noise-only
  protocol (identical RNG seeds) used in `6e821fa` and `d40b4cd`.

**What constitutes a meaningful result:**

- H₀_D9R2 NOT REJECTED (PASS, FP rate ≤ 6.0%): R2 fix is sufficient; branch ready for
  merge review.
- H₀_D9R2 REJECTED (FAIL, FP rate > 6.0%): R2 fix is insufficient; D-009 requires a
  further revision before merge.

**Defect IDs under observation:** D-009 (OSD false-positive filter — second
implementation attempt, R2).

## Section 5 — Recommendations

**D-009 (OSD false-positive filter) — R2 fix is insufficient. Branch
`fix/d009-fp-callsign-filter` CANNOT MERGE at this SHA.**

The R2 fix (native correlation gate at `OSD_CORR_THRESHOLD = 0.10` + Gap A/C text
extensions) reduced the event FP rate from 133.3% (`d40b4cd`, R1) to 75.0% — real
progress — but remains a factor of ~12 above the 6.0% gate threshold. QA analysis of
`S5_matched.csv` (9 FP events / 12 slots) identified two distinct residual categories:

### Gap analysis

| Category | Count | Example | Why it passes | Addressable at text level? |
|---|---|---|---|---|
| **A** — 4-token anomalies: `IsPlausibleMessage` accepts all 4+ token messages unconditionally except the CQ-grid form | 5 of 9 | `KR9NHK/P MR1I O/P QG94` (`CALLSIGN CALLSIGN R GRID` — not a valid Type 1 form) | FT8 Type 1 is always exactly 3 tokens; the only valid 4-token form is `CQ <modifier> <CALLSIGN> <GRID>`. No `token0 == "CQ"` check existed on the 4-token branch. | **Yes** — reject any 4-token message whose token0 ≠ `CQ` |
| **B** — structurally valid 3-token messages | 3 of 9 | `0N4KV 7F4VSB JK98` | Valid callsign lengths, valid portable suffixes, grid letters within [A–R]. Textually indistinguishable from real traffic. | **No** — native gate must be raised |
| **C** — `CQ <...>` (hashed-callsign CQ) | 1 of 9 | `CQ <...>` | Legitimate FT8 Type 4 form; textually indistinguishable from a real CQ-to-hashed-callsign. | **No** — native gate must be raised |

Fixing Category A alone was projected to leave 4 events / 12 slots = 33.3% — still far
above threshold, confirming (again) that the native correlation gate, not further text
rules, carries the bulk of the required suppression.
