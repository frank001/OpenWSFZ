## ADDED Requirements

### Requirement: Bandlimited noise uses a windowed FIR lowpass filter
When `noise_cutoff_hz` is supplied, the synthesiser SHALL bandlimit the AWGN noise vector using a windowed FIR lowpass filter (Kaiser window, ≥ 255 taps) rather than a frequency-domain brickwall zero-out. The FIR filter SHALL be applied via linear convolution with `mode='same'` so that the output length is identical to the input length.

#### Scenario: FIR filter replaces brickwall — no Gibbs ridge
- **WHEN** bandlimited noise is generated with `noise_cutoff_hz=4000` at 48 kHz sample rate
- **THEN** the noise PSD has no spectral peak at 4000 Hz; the PSD at 4000 Hz SHALL be at least 30 dB below the mean passband level

#### Scenario: Output length is preserved
- **WHEN** `add_awgn` is called with `noise_cutoff_hz` set on an input signal of length N
- **THEN** the returned array has exactly N samples

### Requirement: In-band noise PSD is flat within ±1 dB
The bandlimited noise PSD SHALL be flat (≤ ±1 dB peak deviation from the band mean) across the frequency range 100 Hz to 85% of `noise_cutoff_hz`. Flatness is measured using Welch's method with a segment length of at least 4096 samples.

#### Scenario: Passband flatness — 4 kHz cutoff
- **WHEN** noise is generated with `noise_cutoff_hz=4000`, `sample_rate_hz=48000`, duration ≥ 1 s
- **THEN** the Welch PSD from 100 Hz to 3400 Hz has a peak-to-mean deviation of ≤ 1 dB

#### Scenario: Passband flatness — 3 kHz cutoff
- **WHEN** noise is generated with `noise_cutoff_hz=3000`, `sample_rate_hz=48000`, duration ≥ 1 s
- **THEN** the Welch PSD from 100 Hz to 2550 Hz has a peak-to-mean deviation of ≤ 1 dB

### Requirement: `add_noise` accepts `noise_cutoff_hz`
The high-level `add_noise(signal, snr_db, seed, …, noise_cutoff_hz=None)` function SHALL accept an optional `noise_cutoff_hz` parameter and forward it to `add_awgn`. Callers SHALL NOT be required to drop down to `add_awgn` and compute `sigma` manually in order to request bandlimited noise.

### Requirement: In-band SNR is preserved after bandlimiting
Applying `noise_cutoff_hz` SHALL NOT alter the in-band SNR (2500 Hz reference bandwidth). The realised SNR measured by `measure_inband_snr_db` SHALL be within ±0.5 dB of the requested `snr_db` regardless of whether `noise_cutoff_hz` is set.

#### Scenario: SNR preserved with noise cutoff applied
- **WHEN** `add_noise` is called with `snr_db=0.0` and `noise_cutoff_hz=4000` at 48 kHz
- **THEN** `measure_inband_snr_db(signal, result)` returns a value within ±0.5 dB of 0.0 dB

#### Scenario: SNR preserved without noise cutoff
- **WHEN** `add_noise` is called with `snr_db=-10.0` and no `noise_cutoff_hz`
- **THEN** `measure_inband_snr_db(signal, result)` returns a value within ±0.5 dB of −10.0 dB

### Requirement: PSD verification utility is available
A module-level function `verify_noise_psd(noise, cutoff_hz, sample_rate_hz, tolerance_db=1.0)` SHALL be available in `channel.py`. When called with `assert_ok=True`, it SHALL raise `AssertionError` with a descriptive message if the flatness or Gibbs criteria are not met. When called without `assert_ok` (or `assert_ok=False`), it SHALL return `True` if the noise passes and `False` otherwise.

#### Scenario: Verification passes on FIR-filtered noise
- **WHEN** `verify_noise_psd` is called on noise generated by the updated `_lowpass_fir` path with `noise_cutoff_hz=4000`
- **THEN** the function returns `True`

#### Scenario: Verification returns False for unfiltered white noise (regression guard)
- **WHEN** `verify_noise_psd` is called on unfiltered Gaussian white noise (no bandlimiting applied) with `cutoff_hz=4000` at 48 kHz
- **THEN** the function returns `False`; the PSD at 1.2× cutoff (4800 Hz) has ≈ 0 dB attenuation vs the passband mean, well below the required 30 dB threshold

> **Implementation note (2026-06-08):** The original scenario called for testing against "brickwall (FFT bin-zeroing) filtered noise". That premise is physically incorrect: FFT bin-zeroing applied to Gaussian random noise produces flat passband PSD and zero stopband PSD — `verify_noise_psd` would return `True` for it, making it useless as a rejection guard. The Gibbs-phenomenon ridge is a time-domain artefact of the sinc impulse response; it does not appear as a detectable feature in Welch-estimated PSD of random noise. Unfiltered white noise is the correct and deterministic negative test vector. See `test_verify_noise_psd_rejects_unfiltered_white_noise` in `test_channel.py` for the full rationale.

### Requirement: `scipy` is a declared dependency of the R&R study toolchain
`qa/rr-study/requirements.txt` SHALL list `scipy>=1.13` so that `scipy.signal.firwin` and `scipy.signal.fftconvolve` are available in any environment that installs the study dependencies.

#### Scenario: requirements.txt contains scipy
- **WHEN** `qa/rr-study/requirements.txt` is read
- **THEN** it contains a line matching `scipy>=1.13`
