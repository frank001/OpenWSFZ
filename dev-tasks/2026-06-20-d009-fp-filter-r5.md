## 1. Context

D-009: OSD (shim 20260025) extended the decoder floor to ~−27 dB but manufactures CRC-14
false-positive codewords from pure noise (S5: up to 75 % of slots at threshold 0.10).

The R4 approach — keep raising `OSD_CORR_THRESHOLD` until S5 is clean — **cannot succeed**.
The architectural review proved the two targets are mutually exclusive on that single knob:
0 FP on S5 needs ≥ 0.40; preserving S7 co-channel decode needs ≤ 0.35. **Do not continue the
R4 calibration loop.**

Root cause: `corr/norm` is **magnitude-weighted**, so a genuine weak decode (tiny LLRs) and a
noise coincidence are indistinguishable on that one axis. The fix is a **second, orthogonal
discriminant** — `nhard`, the hard-decision Hamming distance between the OSD codeword and the
channel hard decisions. This is the metric WSJT-X uses (`nharderrors`, `osd174_91.f90`).
Genuine decodes are Hamming-close *regardless of SNR*; noise coincidences sit near 87/174.

**Strategy:** implement the cheap, no-ABI-change `nhard` gate + global text rules first.
Measure. Only break the struct ABI (origin flag) if residual FPs remain.

### Action 2 — Tier 1: add the `nhard` gate in `decode.c`

**File:** `native/ft8_lib_build/patched/ft8/decode.c`

The OSD confidence gate appears at **two sites**: `ftx_decode_candidate` (~line 628) and
`ftx_decode_candidate_ap` (~line 800). Both already compute `osd_corr` / `osd_norm` from
`plain174[]` and `llr_for_osd[]` in a single loop. Add an `nhard` accumulation to that **same
loop** and a rejection test **before** the existing `corr/norm` test, at **both** sites.

2.1 — Add the threshold define near the existing one (line 31):

```c
#define OSD_CORR_THRESHOLD 0.10f      /* keep ≤ 0.35; nhard now carries noise rejection      */
#define OSD_NHARD_MAX      <calibrated>  /* see Action 3; reject if codeword too far from LLRs */
```

2.2 — At each gate site, extend the existing loop and add the test. Hard-decision convention
must match the existing `hard_pm1` sign rule (`plain==0` ↔ `llr>0` is agreement):

```c
float osd_corr = 0.0f;
float osd_norm = 0.0f;
int   nhard    = 0;                                   /* NEW */
for (int i = 0; i < FTX_LDPC_N; ++i) {
    float hard_pm1 = (plain174[i] == 0) ? 1.0f : -1.0f;
    osd_corr += hard_pm1 * llr_for_osd[i];
    osd_norm += fabsf(llr_for_osd[i]);
    int hd = (llr_for_osd[i] > 0.0f) ? 0 : 1;         /* NEW: channel hard decision   */
    if (plain174[i] != hd) ++nhard;                   /* NEW: codeword vs channel      */
}
if (nhard > OSD_NHARD_MAX)                             /* NEW: orthogonal reject        */
    return false;
if (osd_norm > 0.0f && (osd_corr / osd_norm) < OSD_CORR_THRESHOLD)
    return false;
```

2.3 — Keep `OSD_CORR_THRESHOLD` at **0.10** for now (or the lowest S7-safe value found in
Action 3). The point of `nhard` is that `corr/norm` no longer has to do the noise rejection.

2.4 — Update the comment block above the define to document the two-feature gate and replace
the stale "calibrated at 0.10; tune if S7 false-negatives appear" inline comment.

> **Diagnostic aid for Action 3:** temporarily log `nhard` for every OSD hit (accepted *and*
> rejected) via the existing TLS/debug logging path so the calibration in Action 3 can build
> the two `nhard` histograms (S5 noise vs S7 genuine). Remove or gate the log before commit.

## 5. Decisions made and why

**D1 — Replace the R4 calibration loop with an `nhard` gate.** The single-knob loop is provably
ceilinged. `nhard` is orthogonal to `corr/norm` (magnitude-independent), free to compute, the
only lever that can touch Category E, and is the reference (WSJT-X) metric.

**D2 — Defer the ABI break behind a measurement gate.** The struct/origin-flag change (review's
Path 1) is taken only if Tiers 1+3 leave residual FPs. No speculative ABI cost.

**D3 — No band-specific text rules.** `/R` and `/P` handling rides the origin flag (Action 5),
keeping the decoder band-agnostic. (Answers review question 3.)

**D4 — Reject Path 2 (SNR gate).** The co-channel decodes OSD rescues sit at ~0 dB, not low SNR;
an SNR gate mis-models the defect. (Answers review question 2.)

**D5 — Widen S5 to ≥ 100 slots.** "0 / 12" gives only a ~22 % upper bound on P(FP) (rule of
three). A meaningful "matches WSJT-X" claim needs the larger sample. (Answers review question 4.)
